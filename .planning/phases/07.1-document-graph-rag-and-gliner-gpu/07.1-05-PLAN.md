<objective>
Rebuild the GLiNER sidecar image so it CAN run on GPU: a multi-stage Dockerfile that compiles
`fast_gliner==0.2.1` from source with the `cuda` extra, plus the compose GPU reservations that
already exist for `agentmemory-embed`.

Purpose: **CONTEXT.md D-09's parenthetical "(`onnxruntime-gpu`)" is factually wrong.**
`fast_gliner` is a Rust/PyO3 binding around `gline-rs`/`ort` — NOT a Python `onnxruntime` wrapper
(verified against the unzipped pinned wheel; `requires_dist` lists only `huggingface-hub`). The
current `pip install "fast_gliner==0.2.1"` installs a **prebuilt CPU-only wheel**. With that wheel,
`execution_provider="cuda"` silently runs on CPU — a CUDA base image cannot fix it, because the
gate is a **compile-time Cargo feature**, not a runtime one. This plan changes the BUILD MODE.
Output: a CUDA-capable GLiNER image and its GPU compose wiring. Proving the binding actually took
is plan 09's job — this plan makes it *possible*, not *proven*.
</objective>
