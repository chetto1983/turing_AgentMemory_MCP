<objective>
Port `search_documents` onto the fusion shape behind a default-OFF flag, preserving flag-OFF
ranking behavior exactly (D-14).

Purpose: `search_documents` is `blend_hybrid_score` -> `_rerank_documents` — a **different
architecture** from `memory_search`'s `RetrievalCandidate` -> `fuse_rankings` -> `diversify_fused`
-> rerank (`store_search.py:279-314`). A graph channel needs the fusion shape. This plan builds the
shape; plan 13 adds the channel.

**Hard constraint:** with the flag OFF, ranking behavior must be **preserved exactly**. The E2E
score gate and Phase 6's locked metrics staying green is the proof.
**The trap, resolved:** `diversify_fused(max_per_source=1)` is correct for memories (one hit per
memory) but would **cap each document at a single chunk**. Documents need `max_per_source > 1`.
This is a real trap, not a detail — it degrades quality silently, with no error.
Output: a flag-gated fusion-shaped document search whose OFF path is provably unchanged.
</objective>
