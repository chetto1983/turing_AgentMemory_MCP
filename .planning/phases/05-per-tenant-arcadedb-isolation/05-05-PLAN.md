<objective>
Build the immutable tenant view/router layer: explicit shared dependency assembly,
per-database single flight, bounded LRU/idle TTL reuse, active-reference safety, and
pseudonymous non-provisioning diagnostics (D-09 and D-12 through D-15).

Purpose: Physical isolation fails if one global store mutates `client` between concurrent
calls, and availability fails if one slow tenant holds a global lock. The router is the
single explicit boundary every foreground/background operation will consume.
Output: A tested resolver protocol/router and tenant-local store construction seam.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Exact identity/provisioner result to cached tenant store | Routing converts untrusted request identity into a long-lived database-bound capability. |
| Concurrent request threads to router state | Multiple tenants contend for cache and first-use lifecycle coordination. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-05-01 | Elevation of Privilege | mutable/global store client | high | mitigate | Task 2 creates immutable tenant-bound stores and tests no client reassignment plus distinct tenant-local state. |
| T-05-05-02 | Information Disclosure | cache keys/status/repr | high | mitigate | Cache/Future/status use opaque names only; tests reject raw identifier fields and diagnostic leakage. |
| T-05-05-03 | Denial of Service | global provisioning lock/unbounded cache | high | mitigate | Per-key Future, provisioning outside RLock, finite LRU capacity/TTL, and overlap/boundary tests. |
| T-05-05-04 | Tampering | eviction during active operation | medium | mitigate | Frozen view and reference-counting semantics; eviction removes only the cache reference and active-reference tests continue successfully. |
| T-05-05-05 | Repudiation | damaged tenant affecting global status | medium | mitigate | Layered content-free status separates router/base readiness from tenant-specific opaque diagnostics; no implicit provision. |
</threat_model>
