<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public store resource ID to ArcadeDB query builder | Caller-controlled IDs can target records that belong to another tenant. |
| Tenant-bound database client to record/edge predicates | A routing bug or wrong client must be contained by record-level scope. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-03-01 | Elevation of Privilege | stable-ID get/update/delete and staging paths | high | mitigate | Tasks 2/3 pair every ID with the exact bound tenant; catalog and focused tests reject ID-only builders. |
| T-05-03-02 | Information Disclosure | edge source/target subqueries | high | mitigate | Task 2 scopes both endpoints, preventing a misbound database from linking or revealing foreign records. |
| T-05-03-03 | Tampering | rebuild/version/community mutations | high | mitigate | Task 3 adds tenant predicates to staging/version/community operations and passes exact scope from the caller. |
| T-05-03-04 | Repudiation | future unclassified query builder | medium | mitigate | Reflection-backed explicit catalog fails CI for every new unclassified builder; exemptions are narrow and reasoned. |
</threat_model>
