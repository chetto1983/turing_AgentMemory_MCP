<objective>
Integrate tenant routing into production FastMCP assembly and every foreground tool while
preserving direct test injection and separating global server/router health from tenant-local
database health (D-04, D-08, D-12, and D-14).

Purpose: The router is ineffective if tool closures retain the old singleton store or health
probes accidentally provision tenants. This plan moves the full foreground boundary without
changing MCP request/response contracts.
Output: Production env wiring, resolve-once tool adapters, layered health, and focused tests.
</objective>
