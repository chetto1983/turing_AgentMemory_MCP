<objective>
Implement synchronous ready-last provisioning: physical database creation/reconciliation,
idempotent schema bootstrap, immutable manifest verification, registry promotion, and bounded
transient retries (D-06 and D-08 through D-11).

Purpose: Database existence alone is not readiness. The requested tenant operation must never
run against an incomplete, wrong-key, wrong-schema, missing, or silently recreated database.
Output: A fault-tested provisioner plus the narrow ArcadeDB lifecycle/schema extensions it uses.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Privileged provisioner to ArcadeDB server lifecycle API | Application credentials can create top-level physical databases. |
| Local registry to tenant database manifest | Two durable evidence stores must agree before tenant data operations are authorized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04-01 | Tampering | ready manifest/client binding | high | mitigate | Immutable singleton fields are reread and constant-time compared before registry promotion or use; wrong-client/mismatch tests fail closed. |
| T-05-04-02 | Elevation of Privilege | duplicate-create race | high | mitigate | Already-exists never authorizes use; Task 2 reruns schema and exact manifest reconciliation. |
| T-05-04-03 | Information Disclosure | provisioning errors/logs | high | mitigate | Errors expose only opaque database identity/fingerprint; Task 1 captures and rejects raw identifier leakage. |
| T-05-04-04 | Denial of Service | retry storms and concurrent create conflicts | medium | mitigate | Retry classifier, finite attempt/ceiling, exponential backoff+jitter, and deterministic no-retry tests bound work. |
| T-05-04-05 | Repudiation / Data loss | ready database disappears | high | mitigate | Registry-ready plus missing database/manifest is a hard incident; tests prove no silent empty reprovisioning. |
</threat_model>
