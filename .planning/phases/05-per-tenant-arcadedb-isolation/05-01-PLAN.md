<objective>
Define the irreversible tenant database identity contract: exact identifier validation,
domain-separated HMAC-SHA-256 naming, strict naming-key loading, and safe pseudonymous
metadata (D-01 through D-05 and D-07).

Purpose: Every later registry, provisioning, cache, and diagnostic decision keys off this
contract. If identity is transformed or guessable, physical database isolation can route a
caller to the wrong tenant before any query predicate has a chance to protect it.
Output: A small standard-library identity module and behavior-first tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP-supplied `user_identifier` to naming function | Untrusted identity crosses into an irreversible database-routing decision. |
| Environment configuration to HMAC operation | Secret naming-key material enters process memory and determines every database name. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01-01 | Spoofing | `validate_user_identifier` | high | mitigate | Task 1 specifies and Task 2 implements exact code-point semantics with rejection of transformations, controls, surrounding whitespace, and surrogates; distinction tests cover case and Unicode lookalikes. |
| T-05-01-02 | Information Disclosure | database-name derivation | high | mitigate | Full HMAC-SHA-256 under a dedicated explicit key prevents guessing likely emails/usernames; tests assert no raw identity in returned objects or diagnostics. |
| T-05-01-03 | Tampering | naming-key configuration | high | mitigate | Strict base64 and minimum-length validation with no fallback; later startup wiring consumes this fail-closed loader. |
| T-05-01-04 | Denial of Service | adversarial Unicode input | medium | mitigate | Linear category validation rejects unsafe code points before cache, registry, or database access; bounded input already follows existing MCP argument limits. |
</threat_model>
