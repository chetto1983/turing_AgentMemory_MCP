<objective>
Move upload, durable document job, and background worker paths onto the same exact-identity
tenant resolver used by foreground MCP tools (D-02, D-12, D-17, and D-19).

Purpose: Physical isolation can still leak if the async worker retains one tenant store or if
upload/job code silently trims identity before lookup. This plan closes that second execution
path without redesigning the document state machine.
Output: Exact durable ownership and per-job tenant-store resolution with concurrency coverage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP upload/job identity to local SQLite/filesystem | Untrusted identity controls durable ownership and staged bytes. |
| Claimed durable job to StoreResolver | Background thread turns persisted tenant identity into a database-bound capability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-07-01 | Spoofing | `.strip()`-transformed upload/job identity | high | mitigate | Task 1 central validator preserves exact values and rejects transformations before mutation; case/Unicode tests. |
| T-05-07-02 | Elevation of Privilege | process-global worker store cache | high | mitigate | Task 2 resolves each claimed job once from its exact persisted identifier; concurrent A/B/C recording tests prove separation. |
| T-05-07-03 | Information Disclosure | cross-tenant job/upload ownership errors | high | mitigate | Non-enumerating errors/absence behavior and foreign-ID tests expose neither stored tenant nor resource existence. |
| T-05-07-04 | Tampering | idempotency/session collisions | high | mitigate | Exact tenant is part of existing hashes/ownership and no longer normalized; collision-prone identical payload tests remain distinct. |
| T-05-07-05 | Denial of Service | tenant failure poisons worker cache | medium | mitigate | No tenant-store cache in manager; each next job resolves independently while existing bounded job retry policy remains. |
</threat_model>
